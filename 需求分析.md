# 一、NovaChat（星聊）V1.0 项目标语

> **一款类似微信的轻量级即时通讯（IM）系统（单体版本），支持单聊、群聊、好友关系、基础资料、消息同步。后续可扩展 AI 助手。**

------

# 二、IM 的核心功能需求（第一阶段专注 IM，不做 AI）

# **1. 用户系统（User System）**

### **1.1 账号管理**

- 账号密码
- 登录注册（账号+ 密码）
- Cookie + Session 鉴权

### **1.2 个人资料**

- 昵称
- 头像
- 个性签名

------

# **2. 好友系统（Social / Contact System）**

这是 IM 的基础。

### **2.1 好友关系维护**

- 添加好友（请求、同意、拒绝）
- 删除好友
- 拉黑好友（黑名单）

------

# **3. 单聊（One-on-One Chat）**

IM 的核心模块。

### **3.1 消息类型**

- 文本消息
- 表情（可直接按文本处理）
- 图片
- 文件（可后期加）
- 语音（可后期加）

### **3.2 消息的存储与传输**

- WebSocket 实时消息推送
- 发送消息的接口
- 消息落库（MySQL + Redis）

### **3.3 聊天记录**

- 获取聊天记录（分页）
- 按时间倒序查询

------

# **4. 群聊系统（Group Chat）**

你将来对标微信，群聊一定要做。

### **4.1 群管理**

- 创建群
- 解散群
- 修改群头像、群名
- 添加成员 / 踢出成员
- 群主 / 管理员权限（可扩展）

### **4.2 群聊消息**

- 同单聊消息结构
- 支持 @功能（可后期扩展）

------

# **5. 会话列表（Conversation List）**

微信顶部看到的会话列表。

- 最近联系人
- 会话置顶
- 会话免打扰（后期扩展）
- 未读消息、未读数统计（依赖 Redis）

------

# **6. 消息同步与多端登录**

这个是 IM 牛逼的核心点，你要重点研究。

### **6.1 多端设备支持**（预留）

- 手机端（Android/iOS）模拟
- Web 端（你先做 Web）

### **6.2 同步机制**

- 客户端登录时“拉取离线消息”
- 多端之间同步未读数、最近会话等信息

------

# **7. 系统架构功能**

你需要后端成熟的架构：

### **7.1 基础能力**

- 鉴权
- WebSocket 服务端
- 文件上传模块（本地 + 对象存储 Minio）
- 日志系统

### **7.2 性能优化方向（可后期加）**

- Redis 缓存用户在线状态
- Redis 计数器维护未读数
- 消息队列（RocketMQ）异步写数据库
